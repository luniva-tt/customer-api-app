Using a `.env` file is better than writing passwords or database credentials directly in code because it improves security and flexibility. Sensitive information such as usernames, passwords, database names, and ports can be stored separately from the application code. This prevents accidental exposure when sharing code through GitHub or other repositories. It also makes it easier to change configurations without modifying the main source code.

Treating the database as a separate backing service is useful because the application and database remain independent from each other. The API can connect to the database service when needed, and the database can be replaced, updated, or scaled without changing the application logic. Docker also makes managing services easier by isolating them inside containers.

Docker improves development and production parity because the same environment is used everywhere. The PostgreSQL container behaves consistently across different systems, reducing errors caused by differences in operating systems, installed software, or configurations. This helps ensure that the application works similarly during development, testing, and deployment.
